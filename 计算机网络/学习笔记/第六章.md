# 多点访问协议
广域网通常采用点对点的链路，也就不存在多点访问控制的问题。

局域网则有多点访问的情况，如果2个或更多站点同时传送，就会产生冲突（collision），多个信号叠加最终导致信号不可用。因而需要用到多路访问协议（介质访问控制协议：MAC ）

MAC解决的就是节点什么时候可以发送、如何使用共享信道。

## 理想的多路访问协议
 1. 当一个节点要发送时，可以R速率发送
 2. 当M个节点要发送，每个可以以R/M的平均速 率发送 
 3. 完全分布的：没有特殊节点协调发送，没有时钟和时隙的同步 
 4. 简单
